# 間取りウォーク — 実務向け 3D 内覧

実住所の地図・外観確認 → 玄関外から入室 → 階段で 2 階まで歩ける Web アプリです。  
物件の登録・保存、間取りトレース（部屋自動分割）、スマホ操作に対応しています。

## 起動（PC）

```bash
cd ~/floorplan-walk
npm install
npm run dev
```

→ http://localhost:5175/

同じWi‑Fiのスマホから触る場合: ターミナルに出る `Network: http://192.168.x.x:5175/` を開く。

## 公開URL（GitHub Pages）

`main` に push すると自動デプロイされます。

```
https://startone-1.github.io/floorplan-walk/
```

## 機能一覧

### 1. 実住所・実物件データの登録
- **＋ 物件登録** で住所・価格・設備・3Dテンプレートを登録
- **地図検索**（OpenStreetMap Nominatim）で緯度経度を自動取得
- Google マップ / ストリートビュー埋め込み
- **localStorage 永続化**（ブラウザに保存）
- **JSON エクスポート / インポート**（他端末へ移行）
- 組み込みサンプルは編集不可、ユーザー物件は編集・削除可

### 2. 2 階・階段の歩行
- 戸建て「桜丘の家」は **1F + 2F** の multi-floor データ
- 階段ゾーンを歩くと **高さが連続変化** し、階が切り替わる
- HUD に **1F / 2F** 表示、ミニマップは現在階

### 3. スマホ用仮想ジョイスティック
- タッチ端末で自動表示
- **左スティック = 移動**、**右スティック = 視点**
- PC は WASD + マウス（従来どおり）

### 4. 間取りトレース（実務向け）
- 画像アップロード
- **壁 / ドア開口 / スケール校正** モード
- 軸スナップ・端点吸着
- **部屋自動分割**（グリッド + フラッドフィル）
- 部屋名の手動編集 → 3D 化 → 物件として保存

## 操作

| 操作 | PC | スマホ |
|------|----|--------|
| 移動 | WASD | 左スティック |
| 見回し | マウス | 右スティック |
| 俯瞰 | B / ボタン | 俯瞰ボタン |
| 地図 | 地図・外観 | 同左 |

## データ構造

```
src/
  store.js              物件 CRUD・ジオコード・永続化
  floorplans/
    apartment1ldk.js    1LDK
    house1f.js          戸建て1F
    house2f.js          戸建て2F
    houseBuilding.js    multi-floor 結合
  segment.js            部屋自動分割アルゴリズム
  tracer.js             トレース UI
  mobile.js             仮想ジョイスティック
  builder.js            3D 生成（複数階・階段）
  core.js               衝突・階段サンプリング
```

### 物件の 3D 紐付け
- `planTemplateId`: `apt-1ldk` | `house-2f`
- `tracedPlanId`: トレース保存 ID
- または `floorPlan` を直接保持

## 本番利用の注意

1. **住所・価格**は必ず物件登録で実データを入力
2. ジオコーディングは Nominatim の利用ポリシーに従ってください（大量リクエスト禁止）
3. 地図の安定表示には Google Maps Embed API キーを推奨
4. トレースは「閉じた壁」が精度の鍵。スケール校正を必ず行う
5. 永続化はブラウザ localStorage。本番は API / DB への置き換えを想定
